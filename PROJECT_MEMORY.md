## 1. Автомобиль (КРИТИЧНО: это НЕ Ram 1500 пикап!)
- **Модель:** 2014 RAM 1500 ProMaster Cargo, колёсная база 136", высокая крыша (High Roof).
- **VIN:** см. локальный `vehicle-private.md` (в `.gitignore`, не публикуется)
- **Двигатель:** 3.6L V6 24V VVT (бензин)
- **КПП:** 6-ступенчатый автомат 62TE
- **Помнить всегда:**
  - ProMaster 1500 — перелицованный Fiat Ducato (L2H2). **Информацию о Ram 1500 (пикап) НЕЛЬЗЯ переносить на ProMaster** — это совершенно разные машины.
  - Кузов **несущий (unibody/monocoque)**, не рамный.
  - Машина — одновременно транспорт и **постоянное жильё** (full-time). Приоритеты: надёжность, автономность, комфорт важнее цены.
  - Системный подход: всё проверяется по первоисточникам (мануалы, форум, видео производителя).

## 3. Электрооборудование (ВАЖНО: пока НИЧЕГО не установлено — всё на этапе выбора/плана)

**Выбрано и подтверждено:**
- **Головное устройство:** XISEDO 9", Android 13 (4/64 ГБ), ProMaster-специфичная CAN-bus интеграция; поддержка нескольких камер, TPMS, OBD2. SIM **T-Mobile** уже куплена.
- **Зеркало-видеорегистратор:** RoadWitness 12", 3-канальный, Sony STARVIS 2 IMX675. Даёт ADAS, частично заменяет отдельный регистратор.
- **Навигационный дисплей:** широкоформатный 11.5" CarPlay/Android Auto (Amazon ASIN **B0GLFSD2DD**), ставится прямо перед водителем (минимум перекрытия обзора).
- **Контроль слепых зон (BSD):** комплект EchoMaster **FCTP-DP1504** (ProMaster-специфичные накладки зеркал с камерами) через коммутатор PAC **VS41**, срабатывание по поворотникам. (Альтернатива в проработке: радар EASYGUARD **EBS001** — монтируется ВЕРТИКАЛЬНО, большой плоской гранью назад/наружу, за пластиком бампера.)
- **OBD-диагностика:** адаптер **OBDLink MX+** + AlfaOBD (дилерский уровень) + Torque Pro (живой дашборд на XISEDO) + OBD Fusion. Для 2014 г. **обход Security Gateway не нужен** (SGW появился с 2018).
- **TPMS:** Tymate **TM7** с внешними датчиками.
- **Парктроники:** EKYLIN, 8 ультразвуковых датчиков, вывод по RCA на экран головного устройства (без отдельного дисплея).
- **Конфликт OBD-порта:** Y-сплиттер (~$10–15), чтобы делить порт между OBDLink MX+ и ADAS-регистратором.

**Открытые вопросы по электрике:**
- **LTE-антенна (2x2 MIMO):** ждём подтверждения, сколько SMA-портов у XISEDO (один или два), прежде чем покупать.

## 4. Структурные/кузовные работы
- **Дверной ограничитель (door check):** Mopar **68169379AE**. Точка крепления треснула, но металл цел. Ремонт: усиливающая пластина (Support Plate) **68367425AA**, заводится через отверстие динамика и заводское овальное технологическое отверстие у A-стойки, фиксируется U-образным кронштейном; трещину сначала проклеить эпоксидкой. На ProMaster ограничитель держит ВСЮ нагрузку открытой двери (петли не дают стопора) — известное слабое место.
- **Пол:** слоёный пирог **XPS-пена -> фанера -> финишное покрытие** (против конденсата/плесени/ржавчины). Ковёр прямо на металл класть нельзя — копит конденсат.

## 5. 3D-модель салона (FreeCAD)
**Цель:** модель интерьера с точностью до мм для раскроя панелей; шкафы по кривым стенкам.

**Окружение / тулчейн:**
- Ubuntu 26.04 (KDE/X11), ноутбук Dell M6800. Каталог проекта: **`~/van-build`**. На 2026-06-15 `git status` из этой папки отвечает `not a git repository` — проверить/инициализировать git отдельно.
- **FreeCAD: AppImage 1.1.1** (x86_64, py311), лежит в **`~/Applications/FreeCAD.AppImage`**, ярлык в меню `~/.local/share/applications/freecad.desktop` (Terminal=false).
  - Ушли со snap: snap блокировал `/run/spnav.sock` (3D-мышь не видна) и подменял `$HOME` (ломало MCP-workflow). Flatpak тоже отвергли (та же песочница). AppImage — без песочницы, работает и 3D-мышь, и MCP.
- **Модель:** `~/van-build/van_v0.py` -> `van_v0.FCStd`. Числовые размеры SEED остаются заглушками; реальная форма кузова теперь добавлена отдельным STL/reference-слоем.
  - 2026-06-15: `van_v0.py` детализирован и пересобран через `~/Applications/FreeCAD.AppImage --console van_v0.py`; текущий `van_v0.FCStd` содержит 108 Part-объектов и 1 Mesh-объект.
  - Добавлено как SEED/reference: арочные колёсные арки вместо прямоугольных, горизонтальные рейки стен (`WallRail_*`, с разрывом на +Y по сдвижной двери), поперечины пола (`FloorCrossmember_*`), верхние продольные рейки (`RoofLongitudinal_*`), зоны хода задних/сдвижной дверей, грубая reference-кабина и контрольные поперечные сечения `Measurement_Sections`.
  - Overall bbox текущего документа с full-body STL/reference-зонами: примерно **5562 × 2074 × 2102 мм**; длина/высота теперь включают внешний body-only STL, задние двери наружу по -X и кабину по +X.
- **Система координат:** origin = пол на пороге ЗАДНИХ дверей по центральной оси. X = длина (0 сзади, +X к кабине), Y = ширина (0 по центру, +Y к пассажиру), Z = высота (0 пол, +Z вверх).

- После перехода snap -> AppImage **аддон нужно переустановить** в `~/.local/share/FreeCAD/Mod/` (в snap-каталоге он стёрся при `snap remove freecad`). Файлы `van_v0.*` в `~/van-build` не пострадали.
- [Точные детали — репозиторий аддона, имя команды запуска RPC-сервера, адрес/порт и блок в конфиге Claude Code — переустановить и зафиксировать ЗДЕСЬ при возобновлении моделирования. Claude Code: впиши реальные значения из системы.]

**Поиск готовой 3D-модели/скана — итог:**
- Точного бесплатного твердотельного CAD ProMaster НЕ существует.
- Найдено и скачано: модель GrabCAD «136 HR camper» — это **SketchUp `.skp` (меш)**, с мебелью (нары), не его фургон; FreeCAD `.skp` не импортирует. Локальные файлы: `~/van-build/Ducato+L2H2+lit+superpose2.skp` и ZIP `~/van-build/fiat-ducato-ram-promaster-136-wheel-base-high-roof-camper-conversion-1.snapshot.1.zip`; ZIP содержит тот же `.skp`.
- 2026-06-15: попытка извлечь из `.skp` текстовые метаданные (`file`, `strings`, `rg -a`) не дала надёжной измеряемой геометрии: видны только бинарные SketchUp-сущности/материалы, но не пригодные координаты рёбер/отверстий. Без SketchUp-импортера считать этот файл источником точных CAD-данных нельзя.
- 2026-06-15: использован экспортированный/полученный STL `~/van-build/Ducato+L2H2+lit+superpose2.stl` (binary SketchUp STL, header `SketchUp STL tmpkneDIP`). Вся сцена: 154 589 треугольников, общий bbox около **5836 × 4027 × 2560 мм**; в сцене есть кузов, человек и внутренняя мебель/модель.
- 2026-06-15: из STL выделен крупнейший связный компонент как кузов без человека/мебели: `~/van-build/Ducato_L2H2_body_reference.stl`, 16 283 треугольника, bbox около **5402 × 2068 × 2102 мм**. Y центрируется на ось 0; X/Z оставлены как в STL (пол около Z=0, задняя часть кузова уходит в -X).
- 2026-06-15: `van_v0.py` теперь добавляет `Scan_Reference/STL_Body_Reference` как `Mesh::Feature` и строит `Scan_Reference/STL_Derived_Sections` — реальные поперечные `Part`-полилинии из STL в X-точках `[0, 400, 920, 1440, 1960, 2480, 3000, 3120]`. Это делает `van_v0.FCStd` визуально ближе к реальному ProMaster/Ducato и даёт рабочие контуры для сравнения с SEED-сечениями.
- Ограничение: STL/reference — это меш, не параметрический solid. Для раскроя мебели его можно использовать как форму/контур, но заводские рёбра, отверстия, внутренние крепёжные точки и точные панели всё равно надо переводить в параметрическую геометрию и проверять ручными промерами.
- 2026-06-15: дополнительный web-поиск подходящих 3D-сканов/моделей именно **Ram ProMaster 1500 136 High Roof / Fiat Ducato L2H2** не дал нового пригодного источника. Находки по 159 WB / ProMaster 2500 / длинным Ducato/Boxer/Jumper не подходят для точной CAD-базы. Внешние источники подтверждают только общее родство Ducato и ProMaster и различие wheelbase-вариантов, но не дают точный интерьерный скан.
- Платные модели — рендер-меши, отвергнуты. Сканы чужих машин использовать только как визуальный reference; сверка пары размеров проверяет масштаб, но не локальные рёбра/отверстия.
- **Вывод:** body-only STL теперь полезен как визуальный/контурный reference формы кузова; источник правды для крепежа, рёбер, отверстий и миллиметровых внутренних размеров всё равно реальный фургон, промеряем руками. Текущая детализация `van_v0` — рабочая база для сравнения и дальнейшей параметризации, не финальная производственная геометрия.

## 6. 3D-манипулятор (3Dconnexion SpaceExplorer) в FreeCAD
- Устройство: 3Dconnexion SpaceExplorer, USB **046d:c627**.
- Драйвер: **spacenavd 1.3.1** из apt (`systemctl enable --now spacenavd`), сокет `/run/spnav.sock`.
- В snap-FreeCAD мышь не видна (песочница режет сокет) -> решилось переходом на AppImage.
- **Кнопки настраивать ТОЛЬКО правкой `user.cfg`**, не через диалог Customize -> «Кнопки пространственного манипулятора» (баг FreeCAD **#17812** — путает активную кнопку; «Сброс» тоже багован — **#19366**, не использовать).
  - Файл: `/home/dimitri/.config/FreeCAD/v1-1/user.cfg`, группа `Spaceball/Buttons`.
  - **Нумерация: в файле Button 0-based, в GUI «Кнопка N». Индекс в файле = (Кнопка N) минус 1.**
  - **FreeCAD должен быть ЗАКРЫТ при правке** (он переписывает `user.cfg` при выходе). Сначала бэкап.
  - Имена команд: `Std_ViewFitAll`, `Std_ViewFront`, `Std_ViewTop`, `Std_ViewRight`, `Std_ViewLeft`, `Std_ViewRear`, `Std_ViewBottom`, `Std_ViewIsometric`.
- **Текущий маппинг кнопок** (физическая -> GUI Кнопка -> file Button -> команда):
  - FIT  -> Кнопка 11 -> Button 10 -> `Std_ViewFitAll`
  - T    -> Кнопка 3  -> Button 2  -> `Std_ViewTop`
  - F    -> Кнопка 6  -> Button 5  -> `Std_ViewFront`
  - R    -> Кнопка 5  -> Button 4  -> `Std_ViewRight`
  - L    -> Кнопка 4  -> Button 3  -> `Std_ViewLeft`
  - 2D   -> Кнопка 15 -> Button 14 -> `Std_ViewIsometric`
- Движение: режим «Доминирующий режим» (Dominant) включён.
